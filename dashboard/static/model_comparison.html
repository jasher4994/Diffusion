From Denoiser to Policy

Structural differences between the original DDPM-style supervised model and the GRPO RL fine-tune — same network, completely different training loop.

The model architecture is identical. Both use the same TextConditionedUNet (1-channel 64×64, 256 base channels, sinusoidal time embedding, frozen CLIP ViT-B/32 text conditioning, classifier-free guidance). GRPO does not change a single layer — it changes how the weights are updated. The RL run even starts from the final DDPM checkpoint (text_diffusion_final_epoch_100.pt).

The one-sentence difference

  • DDPM learns how to denoise by imitating a known answer (the exact noise it added), one random timestep at a time.
  • GRPO keeps that exact same network but learns what to draw by trial-and-error: generate, score with a reward, and nudge the whole sampling trajectory toward higher-reward outputs — while a frozen copy of the original model holds it on a KL leash so it doesn't drift into garbage.
  • Supervised diffusion needs a target image; RL only needs a number. That's why GRPO can chase goals (like symmetry) the original training set never labeled — and also why it can reward-hack if the reward is gameable.
